# 贡献指南

感谢参与本书维护。

## 可贡献内容

- 错别字、命令错误和链接失效；
- GitHub、Docker、Kubernetes、Argo CD 等技术更新；
- 更清楚的解释、图示或实验；
- 排版和无障碍阅读改进。

## 提交流程

1. 创建 Issue 描述问题或改进目标。
2. Fork 仓库并创建分支：`docs/简短描述`。
3. 修改正文时同步检查相关示例、图题和目录。
4. 执行：

```bash
python3 scripts/validate.py
```

5. 提交 Pull Request，并说明修改原因、影响章节和验证方式。

## 写作规范

- 使用简体中文和完整句子；
- 第一次出现的缩写应给出完整名称；
- 区分本地实验、学习演示和生产方案；
- 命令必须可复制执行，占位符必须清楚标明；
- 不提交真实密码、Token、证书私钥或内部地址；
- 技术更新优先引用官方文档。
